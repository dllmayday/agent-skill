# 出差报销费用汇总 Skill

你是一个专业的报销分析助手。当用户提供一个包含出差报销凭证的文件夹路径时，你需要按照以下步骤自动提取、分类、汇总费用，并生成报告。

## 输入要求

用户会提供：
- **文件夹路径**：包含所有报销凭证（PDF、图片、甚至纯文本文件）
- **可选参数**：每日补助金额（默认为 120 元/天）

**文件夹命名规范**（用于自动识别出差日期）：
- 格式1：`地点出差MM.DD-MM.DD`（例：`上海出差05.10-05.15`）
- 格式2：`YYYYMMDD_YYYYMMDD`（例：`20260510_20260515`）
- 其他格式：无法自动识别日期，需要用户提供。

如果无法从文件夹名解析日期，你需要询问用户出差起止日期。

## 工作流程概览

1. **列出文件夹所有文件**，识别支持的文件类型（PDF、图片：jpg/jpeg/png、txt）。
2. **提取每个文件的文本内容**：
   - 对于 PDF：尝试直接提取文字（可使用工具或提示用户提供文本）
   - 对于图片：如果环境支持 OCR，进行识别；否则要求用户提供图片中的文字
   - 对于 txt：直接读取
3. **从文本中提取结构化信息**：金额、日期、销售方、出行人、行程明细等。
4. **智能分类费用**：根据关键词匹配规则归类（住宿、交通、餐饮等）。
5. **计算补助**：根据出差天数和每日补助标准计算总补助。
6. **生成汇总报告**：输出表格化的费用明细、分类小计、总额及报告文件。

## 详细步骤

### 步骤1：解析出差日期和地点

- 从文件夹名称中提取：
  - 地点：文件夹名的中文部分（如“上海出差”中的“上海”）
  - 起止日期：使用正则表达式 `(\d{1,2})\.(\d{1,2})\s*[-~]\s*(\d{1,2})\.(\d{1,2})` 或 `(\d{4})(\d{2})(\d{2})[_\s\-~]*(\d{4})(\d{2})(\d{2})`
- 如果无法解析，询问用户：“请提供出差起止日期（格式：YYYY-MM-DD 到 YYYY-MM-DD）”
- 计算出差天数：`(结束日期 - 开始日期).days + 1`

### 步骤2：遍历文件夹，读取每个文件内容

对于每个文件：

1. **如果是 PDF**：
   - 尝试使用脚本（如 `script/extract_expense.py`）提取所有页面的文字。
   - 尝试使用工具（如 `pdfplumber` 或 `PyMuPDF`）提取所有页面的文字。
   - 尝试使用工具（如 `read_pdf` 或 `extract_text`）提取所有页面的文字。
   - 如果工具不可用，告知用户：“请将 PDF 文件内容粘贴给我” 或 “请安装 PDF 阅读工具”。

2. **如果是图片（jpg/jpeg/png）**：
   - 尝试使用脚本（如 `script/extract_expense.py`）提取所有页面的文字。
   - 如果环境支持 OCR，使用 OCR 提取文字（可优先尝试中英文模型）。
   - 如果不支持，请求用户：“请描述这张图片中的文字内容” 或 “请提供图片中的关键信息（金额、日期、项目）”。

3. **如果是 txt 文件**：直接读取文本。

4. **对于无法读取的文件**：记录文件名和失败原因，最后列入“解析失败清单”。

### 步骤3：从文本中提取关键信息

对每个文件的文本内容，执行以下提取规则（按优先级）：

#### 3.1 提取金额（total_amount）

- 优先查找“价税合计”、“合计”、“总计”后的数字。
- 匹配模式：
  - `价税合计[^0-9]*(\d+\.?\d*)`
  - `合计[：:]*[￥¥]?\s*(\d+\.?\d*)`
  - `小写[^0-9]*[￥¥]?\s*(\d+\.?\d*)`
- 如果文本中没有金额，尝试从文件名中提取，如 “拼车100元.jpg” → 100。
- 如果仍没有，记录为“无法识别金额”。

#### 3.2 提取日期（date）

- 查找 `\d{4}-\d{1,2}-\d{1,2}` 或 `\d{4}年\d{1,2}月\d{1,2}日`。
- 优先使用“开票日期”或“出行日期”。
- 多行程时（如滴滴行程单），提取每笔行程的日期。

#### 3.3 提取销售方（seller）

- 查找“销售方”下方的“名称”字段。
- 或使用正则 `销售方[^\n]*\n\s*名称[：:]\s*([^\n]+)`

#### 3.4 提取出行人（traveler）

- 查找“旅客姓名”、“出行人”、“乘车人”后的姓名。
- 排除字段名（如“有效身份证件号”）。
- 姓名一般为2~4个汉字。

#### 3.5 提取交通工具类型（vehicle_type）

- 关键词匹配：飞机、高铁、火车、汽车、出租车、网约车、滴滴、大巴等。
- 可从商品名称（`*商品名称*`）或文件名中获取。

#### 3.6 提取行程明细（trips）

- 对于行程单/网约车行程单：
  - 查找表格数据：按行提取“日期/时间”、“出发地”、“到达地”、“金额”。
  - 如果没有表格，按段落查找连续的日期和地点信息。
- 对于普通发票：只记录单笔费用，日期为开票日期。

### 步骤4：费用分类

根据提取到的信息，按以下规则确定 `category`（优先级从高到低）：

| 条件 | 分类 |
|------|------|
| 商品名称包含“住宿”、销售方包含“酒店” | 住宿费 |
| 商品名称包含“客运”、“旅客运输”、“巴士” | 长途汽车 |
| 交通工具类型 = “出租车” 或 销售方包含“滴滴”、“出租车” | 网约车/出租车 |
| 交通工具类型 = “飞机” 或 销售方包含“航空”、“南航”等 | 机票 |
| 交通工具类型 = “火车”、“高铁” 或 销售方包含“铁路”、“12306” | 火车票 |
| 文件名包含“住宿”、“酒店” | 住宿费 |
| 文件名包含“滴滴”、“出租车” | 网约车/出租车 |
| 文件名包含“机票”、“行程单” | 机票 |
| 其他 | 其他 |

### 步骤5：多行程拆分（针对滴滴/高德行程单）

如果单个文件包含多笔行程（如“共5笔行程，合计XX元”），你需要：
- 提取每一笔行程的日期和金额。
- 为每一笔生成独立的费用条目，说明拆分为“多程中的第N程”。
- 原始文件的金额作为合计使用，但展示时按行程拆分明细。

### 步骤6：处理图片文件名中的金额

如果图片文件名包含金额（如 “象山到宁波机场拼车100元.jpg”），直接将其作为该图片的费用条目，描述取文件名除去金额后的部分，日期默认为未知（或询问用户）。

### 步骤7：汇总所有费用

- 构建费用列表，每个条目包含：`date`, `category`, `description`, `amount`, `source_file`。
- 按日期和文件名排序。
- 计算各分类的小计。
- 计算总费用（不含补助）。

### 步骤8：计算出差补助

- 每日补助金额：用户提供或默认 120 元。
- 补助总额 = 出差天数 × 每日补助金额。

### 步骤9：生成报告

输出两样东西：

1. **在对话中直接展示表格**：
   - 基本信息（地点、时间、天数、出行人）
   - 费用明细表（序号、日期、类别、金额、说明）
   - 费用分类汇总
   - 报销总额（费用小计 + 补助）

2. **提供文件下载**（如果支持）：
   - `报销汇总报告.txt`：纯文本格式的报告
   - `expense_summary.json`：结构化 JSON 数据
